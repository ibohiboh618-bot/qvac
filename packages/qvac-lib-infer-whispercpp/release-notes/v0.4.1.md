# QVAC Transcription Whisper Addon v0.4.1 Release Notes

This release switches model distribution to downloading standard Whisper models directly from HuggingFace. New download scripts and automatic model fetching in test helpers simplify getting started.

## Breaking Changes

### Removed Legacy Data Loader

The legacy peer dependency and dev dependency for P2P model loading have been removed. All references to P2P-based model loading have been stripped from the test helpers and integration tests.

Models are now expected to be pre-downloaded to disk or automatically fetched from HuggingFace by the test infrastructure.

### Removed Legacy Examples

The following example files have been removed:

- `examples/transcription.hd.js`
- `examples/exampleVad.hd.js`

Use `examples/quickstart.js` or `examples/example.live-transcription.js` instead, which use the filesystem data loader.

## Features

### HuggingFace Model Downloads

Standard Whisper models and the Silero VAD model are now downloaded directly from their public HuggingFace repositories:

- Whisper models from [ggerganov/whisper.cpp](https://huggingface.co/ggerganov/whisper.cpp) (tiny through large-v3-turbo, including quantized variants)
- VAD model from [ggml-org/whisper-vad](https://huggingface.co/ggml-org/whisper-vad) (ggml-silero-v5.1.2.bin)

The test helpers (`ensureWhisperModel` and `ensureVADModel`) now automatically download models from HuggingFace when they are not found locally, using `curl`. This means integration tests will self-provision their required models on first run.

### Download Scripts

A new script is available under `scripts/`:

- **`download-models.sh`** -- Interactive script for downloading standard Whisper models from HuggingFace. Run it via `npm run download-models`. Supports individual model selection, quantized variants, and bundle downloads (e.g., tiny + VAD for development).

### Architecture Documentation

New documentation files have been added covering the internal architecture and detailed data flows of the transcription addon.
