# QVAC Transcription Parakeet Addon v0.1.5 Release Notes

This release removes the Hyperdrive peer dependency, making the package fully usable with only the filesystem data loader. It also adds third-party attribution and parameterizes the S3 model download script for flexible deployment.

## Breaking Changes

### Hyperdrive Dependency Removed

The `@qvac/dl-hyperdrive` package has been removed from both `devDependencies` and `peerDependencies`. Projects that previously relied on the implicit Hyperdrive peer dependency should ensure they use `@qvac/dl-filesystem` (or another compatible loader) instead. No code changes are needed if you were already using the filesystem loader.

### S3 Download Script Requires `MODEL_S3_BUCKET` Environment Variable

The `download-models-s3.sh` script no longer contains a hardcoded S3 bucket. You must now set the `MODEL_S3_BUCKET` environment variable before running the script:

```bash
MODEL_S3_BUCKET=my-bucket ./scripts/download-models-s3.sh --access-key <KEY> --secret-key <SECRET>
```

## Other

### Third-Party Attribution

A `NOTICE` file has been added with full third-party dependency attributions. Both `LICENSE` and `NOTICE` are now included in the published npm package.
