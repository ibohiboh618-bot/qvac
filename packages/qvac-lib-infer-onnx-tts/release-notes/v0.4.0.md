# QVAC TTS ONNX Addon v0.4.0 Release Notes

This release removes the Piper TTS engine and all associated dependencies. The addon now exclusively uses the Chatterbox engine for text-to-speech synthesis. Integration tests have been improved with real reference audio for more accurate voice cloning validation.

## Breaking Changes

### Piper TTS Engine Removed

The Piper TTS engine has been completely removed from the addon, and all Piper-specific configuration parameters are no longer accepted. If you were using Piper, you must migrate to the Chatterbox engine:

**Before (Piper):**
```js
const model = new ONNXTTS({
  mainModelUrl: 'en_US-lessac-medium.onnx',
  configJsonPath: 'en_US-lessac-medium.onnx.json',
  eSpeakDataPath: 'espeak-ng-data'
}, { language: 'en', engine: 'piper' })
```

**After (Chatterbox):**
```js
const model = new ONNXTTS({
  tokenizerPath: 'chatterbox/tokenizer.json',
  speechEncoderPath: 'chatterbox/speech_encoder.onnx',
  embedTokensPath: 'chatterbox/embed_tokens.onnx',
  conditionalDecoderPath: 'chatterbox/conditional_decoder.onnx',
  languageModelPath: 'chatterbox/language_model.onnx',
  referenceAudio: audioFloat32Array
}, { language: 'en' })
```

The following have been removed:
- `PiperTTSArgs` type from the TypeScript definitions
- `ENGINE_PIPER` constant and Piper engine detection logic
- `mainModelUrl`, `configJsonPath`, `eSpeakDataPath` constructor parameters
- `TashkeelDiacritizer` (Arabic diacritization for Piper)
- Piper vcpkg dependencies: `piper`, `piper-phonemize`, `espeak-ng`

The `EngineType` enum is preserved in the C++ layer with only `Chatterbox` for now, to facilitate adding new engines in the future.

## Other

- Removed Piper model downloads from CI integration test and C++ test coverage workflows
- Updated README documentation to reflect Chatterbox-only usage
- Removed Piper-specific examples (`example-onnx-tts.js`, `example-hd-onnx-tts.js`, `example-onnx-ttsGPU.js`, `example.reload.js`)
- Updated the Chatterbox example with automatic reference audio resampling
- Renamed `wav-generator-helper.js` to `wav-helper.js` in examples
- Added `resampleLinear` utility to both `examples/wav-helper.js` and `test/utils/wav-helper.js`
- Consolidated test WAV utilities (`readWavAsFloat32`, `createWavBuffer`, `resampleLinear`) into `test/utils/wav-helper.js` to remove cross-references between `test/` and `examples/`
