# QVAC TTS ONNX Addon v0.5.4 Release Notes

This release adds lazy ONNX session loading to reduce peak memory usage during synthesis, preventing out-of-memory kills on memory-constrained devices. It defaults to true in iOS, and to false in other platforms

## Features

### Lazy ONNX session loading

It is now possible to control the Chatterbox engine to no longer load all four ONNX sessions (embed_tokens, speech_encoder, language_model, conditional_decoder) at once during `load()`. Instead, sessions are created on demand during `synthesize()` and released as soon as they are no longer needed:

- `speech_encoder` is released immediately after its single use in the first iteration.
- `embed_tokens` and `language_model` are released after the token generation loop.
- `conditional_decoder` is loaded only after the other sessions are released, so it is never co-resident in memory with them.

This behavior is controlled by the `lazySessionLoading` configuration option, which can be set from JavaScript at initialization time. It defaults to `true` on iOS (`platform() === 'ios'`) and `false` on all other platforms.

```js
const tts = new ONNXTTS({
  // ... model paths ...
  lazySessionLoading: true // override the platform default
}, config)
```
