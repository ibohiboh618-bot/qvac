---

```ts
function defineDuplexHandler(definition: DuplexPluginHandlerDefinition<TRequest, TResponse>): PluginHandlerDefinition<TRequest, TResponse>;
```

## Parameters

| Name | Type | Required? | Description |
| --- | --- | :---: | --- |
| definition | `DuplexPluginHandlerDefinition<TRequest, TResponse>` | ✓ | The duplex plugin handler definition (request/response schemas and streaming handler function). |

### `DuplexPluginHandlerDefinition`

| Field | Type | Required? | Description |
| --- | --- | :---: | --- |
| requestSchema | `TRequest` | ✓ | Zod schema for validating incoming requests |
| responseSchema | `TResponse` | ✓ | Zod schema for validating outgoing responses |
| streaming | `true` | ✓ | Must be `true` — duplex handlers are always streaming |
| duplex | `true` | ✓ | Must be `true` — marks this handler as bidirectional |
| handler | `TRequest extends ZodType<infer I, unknown, $ZodTypeInternals<infer I, unknown>> ? TResponse extends ZodType<infer O, unknown, $ZodTypeInternals<infer O, unknown>> ? (request: I, inputStream: AsyncIterable<...>) => AsyncGenerator<...> : never : never` | ✓ | The handler function — receives a validated request and an input stream, yields validated response chunks |

## Returns

`PluginHandlerDefinition<TRequest, TResponse>` — The same definition object, with full type inference applied. This is an identity function used for type checking.
