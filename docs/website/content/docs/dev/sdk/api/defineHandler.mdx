```ts
function defineHandler(definition: PluginHandlerDefinition<TRequest, TResponse>): PluginHandlerDefinition<TRequest, TResponse>;
```

## Parameters

| Name | Type | Required? | Description |
| --- | --- | :---: | --- |
| definition | `PluginHandlerDefinition<TRequest, TResponse>` | ✓ | The plugin handler definition (request/response schemas and handler function). |

### `PluginHandlerDefinition`

| Field | Type | Required? | Description |
| --- | --- | :---: | --- |
| requestSchema | `TRequest` | ✓ | Zod schema for validating incoming requests |
| responseSchema | `TResponse` | ✓ | Zod schema for validating outgoing responses |
| streaming | `boolean` | ✓ | Whether this handler uses streaming responses |
| duplex | `boolean` | ✗ | — |
| handler | `TRequest extends ZodType<infer I, unknown, $ZodTypeInternals<infer I, unknown>> ? TResponse extends ZodType<infer O, unknown, $ZodTypeInternals<infer O, unknown>> ? (request: I, inputStream?: AsyncIterable<...> \| undefined) => Promise<...> \| AsyncGenerator<...> : never : never` | ✓ | The handler function — receives validated request, returns validated response |

## Returns

`PluginHandlerDefinition<TRequest, TResponse>` — The same definition object, with full type inference applied. This is an identity function used for type checking.
