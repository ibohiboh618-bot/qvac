---
title: "finetune( )"
titleStyle: code
description: Starts, resumes, inspects, pauses, or cancels a finetuning job for a loaded model.
---

```ts
function finetune(params: FinetuneRunParams, rpcOptions?: RPCOptions): FinetuneHandle;
function finetune(params: FinetuneReplyParams, rpcOptions?: RPCOptions): Promise<FinetuneResult>;
```

## Parameters

| Name | Type | Required? | Description |
| --- | --- | :---: | --- |
| params | `FinetuneRunParams` | ✓ | The finetuning parameters |
| rpcOptions | [`RPCOptions`](#rpcoptions) | ✗ | Optional RPC transport options |

### `params`

| Field | Type | Required? | Default | Description |
| --- | --- | :---: | --- | --- |
| modelId | `string` | ✓ | — | The identifier of the loaded model to finetune |
| operation | `"start" \| "resume"` | ✗ | — | The finetuning operation. Omit it to let the add-on choose whether to start fresh or resume automatically |
| options | `\{ assistantLossOnly?: boolean; batchSize?: number; chatTemplatePath?: string; checkpointSaveDir?: string; checkpointSaveSteps?: number; contextLength?: number; learningRate?: number; loraAlpha?: number; loraInitStd?: number; loraModules?: string; loraRank?: number; loraSeed?: number; lrMin?: number; lrScheduler?: "constant" \| "cosine" \| "linear"; microBatchSize?: number; numberOfEpochs?: number; outputParametersDir: string; trainDatasetDir: string; validation: \{ type: "none" \} \| \{ fraction?: number; type: "split" \} \| \{ path: string; type: "dataset" \}; warmupRatio?: number; warmupRatioSet?: boolean; warmupSteps?: number; warmupStepsSet?: boolean; weightDecay?: number \}` | ✓ | `finetuneOptionsPayloadSchema` | Finetuning options for run and `getState` operations |

### `rpcOptions`

| Field | Type | Required? | Description |
| --- | --- | :---: | --- |
| forceNewConnection | `boolean` | ✗ | When `true`, skip any cached RPC connection and open a fresh one for this call. |
| healthCheckTimeout | `number` | ✗ | Timeout in milliseconds for the health-check probe that precedes the RPC call. |
| profiling | `\{ enabled?: boolean; includeServerBreakdown?: boolean; mode?: "summary" \| "verbose" \}` | ✗ | Per-call profiler configuration; when present, overrides the SDK's global profiler settings for this request. |
| timeout | `number` | ✗ | Per-call RPC timeout in milliseconds; overrides the SDK-level default for this request only. |

## Returns

The return type depends on the `operation`:

**Run overload** (operation omitted, `"start"`, or `"resume"`):

`FinetuneHandle` — Object with the following fields:

### `FinetuneHandle`

| Field | Type | Description |
| --- | --- | --- |
| progressStream | `AsyncGenerator<\{ accuracy: number \| null; accuracy_uncertainty: number \| null; current_batch: number; current_epoch: number; elapsed_ms: number; eta_ms: number; global_steps: number; is_train: boolean; loss: number \| null; loss_uncertainty: number \| null; total_batches: number \}>` | Stream of training progress ticks |
| result | `Promise<\{ stats?: \{ epochs_completed: number; global_steps: number; learning_rate?: number; train_accuracy?: number; train_accuracy_uncertainty?: number \| null; train_loss?: number; train_loss_uncertainty?: number \| null; val_accuracy?: number; val_accuracy_uncertainty?: number \| null; val_loss?: number; val_loss_uncertainty?: number \| null \}; status: "IDLE" \| "RUNNING" \| "PAUSED" \| "CANCELLED" \| "COMPLETED"; type: "finetune" \}>` | Resolves when the job finishes |

### `result`

| Field | Type | Required? | Default | Description |
| --- | --- | :---: | --- | --- |
| stats | `\{ epochs_completed: number; global_steps: number; learning_rate?: number; train_accuracy?: number; train_accuracy_uncertainty?: number \| null; train_loss?: number; train_loss_uncertainty?: number \| null; val_accuracy?: number; val_accuracy_uncertainty?: number \| null; val_loss?: number; val_loss_uncertainty?: number \| null \}` | ✗ | — | — |
| status | `"IDLE" \| "RUNNING" \| "PAUSED" \| "CANCELLED" \| "COMPLETED"` | ✓ | `finetuneStatusSchema` | — |
| type | `"finetune"` | ✓ | — | — |

## Throws

| Error | When |
| --- | --- |
| `INVALID_RESPONSE_TYPE` | Response type does not match expected `"finetune"` |
| `STREAM_ENDED_WITHOUT_RESPONSE` | Stream ended without receiving the terminal finetune response |

## Example

```typescript
const handle = finetune({
  modelId,
  options: {
    trainDatasetDir: "./dataset/train",
    validation: { type: "split", fraction: 0.05 },
    outputParametersDir: "./artifacts/lora",
    numberOfEpochs: 2,
  },
});

for await (const progress of handle.progressStream) {
  console.log(progress.global_steps, progress.loss);
}

console.log(await handle.result);

const pauseResult = await finetune({ modelId, operation: "pause" });
console.log(pauseResult.status);
```
